Afinal, o que é Google Amp?

O projeto AMP é uma iniciativa de código aberto com o objetivo de criar uma Web melhor para todos. O projeto possibilita a criação de websites e anúncios rápidos, atraentes e de alto desempenho em vários dispositivos e plataformas de distribuição.

Mas por quê usar AMP?

A velocidade de uma página da Web melhora a experiência do usuário e as principais métricas de negócios. As páginas AMP são carregadas quase instantaneamente. Isso permite oferecer uma experiência rápida e consistente em todos os dispositivos e plataformas com links para páginas AMP, incluindo Google, Bing, LinkedIn e muito mais. Esses ganhos de desempenho geralmente causam melhorias em valores importantes, como o tempo gasto na página, as visitas de retorno e a CTR.

Marcação obrigatória

  1. Ser iniciados pelo doctype <!doctype html>.
  2. Conter uma tag <html ⚡> de nível superior (<html amp> também é aceita).
  3. Conter as tags <head> e <body> (elas são opcionais em HTML).
  4. Conter uma tag <link rel="canonical" href="$SOME_URL"> dentro do cabeçalho que aponte para a versão em HTML comum do documento em AMP HTML, ou para o próprio documento se a versão em HTML não existir.
  5. Conter uma tag <meta charset="utf-8"> como primeira filha da tag do cabeçalho.
  6. Conter uma tag <meta name="viewport" content="width=device-width,minimum-scale=1">dentro da tag do cabeçalho. Também é recomendável incluir initial-scale=1.
  7. Conter uma tag <script async src="https://cdn.ampproject.org/v0.js"></script> como o último elemento do cabeçalho (isso inclui e carrega a biblioteca de AMP JS).

Páginas vinculadas com um link<>

Adicione o seguinte à página não AMP:

							
							 <link rel="amphtml" href="https://www.example.com/url/to/amp/document.html">

Adicione isto à página AMP:

							
							 <link rel="canonical" href="https://www.example.com/url/to/full/document.html">

E se eu só tiver uma página?

Se você só tiver uma página, e essa página for AMP, ainda assim deverá adicionar o link canônico a ela, mas ele simplesmente apontará para a própria página:

							
							 <link rel="canonical" href="https://www.example.com/url/to/amp/document.html">

Estilizando os componentes

O estilo e o layout nas páginas AMP em HTML são muito semelhantes aos de páginas HTML normais. Nos dois casos, você usará CSS.

Adicione todos os CSS dentro de uma tag <style amp-custom> no cabeçalho do documento. Por exemplo:

							
							 <head>
								<style amp-custom>
								 body { background-color: white; }
								 amp-img { border: 5px solid black; }
								 amp-img.grey-placeholder { background-color: grey; }
								</style>
							 </head>
IMPORTANTE — Sua página pode ter somente uma tag <style amp-custom>, pois não é permitido ter mais de uma nas AMP.

Defina os estilos de componentes com seletores de classe ou elemento usando propriedades CSS comuns. Por exemplo:

							
							 <amp-img
								class="grey-placeholder"
								src="https://placekitten.com/g/500/300"
								srcset="/img/cat.jpg 640w, /img/kitten.jpg 320w"
								width="500" height="300" layout="responsive">
							 </amp-img>
